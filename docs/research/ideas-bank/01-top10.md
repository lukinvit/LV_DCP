# Топ-10 идей с максимальным ROI

Ранжировано по соотношению **влияние / срок**. Это критический путь.

---

## 1. bge-m3: единый эмбеддер (dense + sparse + multivector)

- **Что даёт простыми словами:** одна модель за один проход даёт три типа векторов — классический (смысл), sparse (точное совпадение ключевых слов) и multivector (пословный). Все три записываем в Qdrant как named vectors одной точки.
- **Какую проблему решает:** наш текущий чистый dense-поиск плохо ловит редкие идентификаторы (`UUID`, `_internal_X`), а также не умеет token-level precision. Три сигнала закрывают разные классы запросов.
- **Где:** `libs/embeddings/bge_m3.py`, миграция Qdrant-коллекций на named vectors.
- **Влияние:** **H** — ожидаемо **+20–30% NDCG@10** на code retrieval по бенчмаркам.
- **Срок:** **2–3 дня**.
- **Источник:** FlagEmbedding (BAAI), Qdrant docs по sparse/multivector.

---

## 2. bge-reranker-v2-m3 на финальной стадии

- **Что даёт:** маленький cross-encoder берёт топ-100 кандидатов из vector+graph и переранжирует в топ-20. Смотрит на пару (query, candidate) целиком, а не только на dot-product.
- **Проблема:** даже идеальный эмбеддинг даёт шум в топ-30. Cross-encoder убирает его за копейки.
- **Где:** `libs/retrieval/rerank.py`.
- **Влияние:** **H** — **+10–25% NDCG@10** поверх любого улучшения эмбеддинга.
- **Срок:** **1–2 дня**.
- **Источник:** FlagEmbedding, Continue.dev (LLMReranker), Cursor (apply model).

---

## 3. Matryoshka + binary quantization в Qdrant

- **Что даёт:** храним компактные 256-битные векторы + полные 1024-dim float рядом. Первичный поиск — по binary (32× быстрее и дешевле по памяти), доранжировка top-50 — по full precision.
- **Проблема:** наш ADR-001 жёстко лимитирует память и cost. На 1M символов full-precision индекс — гигабайты RAM. Binary-слой снимает этот потолок.
- **Где:** конфиг Qdrant-коллекций, `libs/embeddings/quant.py`.
- **Влияние:** **H** — **32× экономия памяти при <2% потерь recall**.
- **Срок:** **1 день**.
- **Источник:** Qdrant docs, Jina embeddings v3.

---

## 4. `watchfiles` вместо `watchdog` в desktop-агенте

- **Что даёт:** Rust-ядро (notify-rs), native async-iterator (`async for changes in awatch(...)`), встроенный debounce, batch-события как атомарный `set`.
- **Проблема:** `watchdog` теряет события после sleep/resume на macOS, деградирует на monorepos 10k+ файлов, требует самописной очереди дебаунса. У нас это ядро Phase 1 — оно должно быть непробиваемым.
- **Где:** `apps/agent/watcher.py`.
- **Влияние:** **H** — убирает класс багов «watcher потерял событие», код сокращается на ~200 строк.
- **Срок:** **2–3 дня** (миграция + регресс-тесты).
- **Источник:** samuelcolvin/watchfiles.

---

## 5. `gitleaks` + `detect-secrets` гибрид (privacy layer)

- **Что даёт:** перед отправкой любого pack в Claude API — двухуровневая проверка секретов. Gitleaks как subprocess для полного сканирования репо на bootstrap, detect-secrets как inline-Python для hot-path каждого запроса.
- **Проблема:** ТЗ §17 требует privacy-mode. Сейчас мы можем случайно отправить в LLM файл с `JWT_SECRET` или API-ключом.
- **Где:** `libs/policies/privacy.py`, `apps/agent/preflight.py`.
- **Влияние:** **H** — закрывает один из двух блокеров production-use.
- **Срок:** **2–3 дня**.
- **Источник:** gitleaks, Yelp/detect-secrets.

---

## 6. RAGAS + promptfoo как eval harness

- **Что даёт:** `make eval` возвращает цифры — Context Precision@K, Context Recall@K, Faithfulness. Promptfoo — YAML-декларации тестов с CI-интеграцией; ragas — метрики как LLM-judges.
- **Проблема:** ADR-002 требует retrieval quality как контракт. Без метрик мы не знаем, стали ли все остальные изменения лучше или хуже.
- **Где:** `libs/eval/metrics.py`, `tests/eval/ragas_suite/`, `Makefile` (`make eval`).
- **Влияние:** **H** — без этого все остальные улучшения слепые.
- **Срок:** **2–3 дня**.
- **Источник:** explodinggradients/ragas, promptfoo/promptfoo.

---

## 7. Personalized PageRank для graph expansion

- **Что даёт:** при graph-стадии retrieval seed-ноды (символы из query) используются как personalization-вектор в PageRank над графом `calls/imports/inherits`. Важные callers/callees всплывают наверх.
- **Проблема:** сейчас graph expansion — грубый BFS по N hops. Даёт много мусора: транзитивных связей, которые формально есть, но смысла не имеют.
- **Где:** `libs/retrieval/graph_expand.py`, `libs/graph/ppr.py`.
- **Влияние:** **H** — качественно меняет «что попадает в pack» при вопросах про конкретные символы.
- **Срок:** **2–3 дня** (networkx даёт PPR out-of-the-box).
- **Источник:** HippoRAG (OSU-NLP-Group), Aider repo-map.

---

## 8. Shadow-git checkpoints для edit pipeline (фаза 2)

- **Что даёт:** каждый шаг агентной правки коммитится в скрытый git-репо в `.context/shadow/`. Откат за одну команду. Пользовательский git не трогается.
- **Проблема:** автоматические правки без checkpoints — потеря работы. ТЗ §16 обязывает «edit guard pipeline».
- **Где:** `libs/impact/shadow_git.py`, `apps/agent/apply.py`.
- **Влияние:** **H** — обязательный foundation для фазы 2, без него нельзя давать агенту write-доступ.
- **Срок:** **1 неделя**.
- **Источник:** Cline, Cursor «shadow workspace».

---

## 9. Search/replace edit format вместо unified diff

- **Что даёт:** агент возвращает `SEARCH:`/`REPLACE:` блоки текста вместо patch-ей. LLM делают меньше ошибок, парсер — 50 строк, конфликт-detection — простой equality-check.
- **Проблема:** unified diff сложно генерировать и сложно парсить; на длинных файлах LLM путают смещения строк.
- **Где:** `libs/impact/edit_format.py`.
- **Влияние:** **H** — критичен для робастности edit pipeline.
- **Срок:** **1–2 дня**.
- **Источник:** Aider (benchmarks показывают заметно выше pass-rate, чем udiff).

---

## 10. Langfuse self-hosted как observability + prompt registry

- **Что даёт:** traces retrieval-пайплайна с полными token/cost метаданными, versioned prompts (можно менять без деплоя), datasets + eval runs с diff между запусками.
- **Проблема:** сейчас observability = structlog. Не видно cost-per-project, нет diff метрик между версиями, prompts размазаны по коду.
- **Где:** `libs/telemetry/langfuse.py`, `libs/summarization/prompts/` → registry, docker-compose service.
- **Влияние:** **M-H** — закрывает ADR-001 budgets как инфраструктуру, а не как код.
- **Срок:** **3–5 дней** (включая self-hosted deploy).
- **Источник:** langfuse/langfuse.
