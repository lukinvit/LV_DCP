# v0.8.66 — Karpathy-aligned wiki prose + per-project wiki in Obsidian

**Date:** 2026-04-25
**Tag:** `v0.8.66`
**Theme:** Wiki quality + Obsidian visibility

## What ships

Three coordinated changes. Two improve the wiki itself, one makes the
per-project wiki visible inside Obsidian alongside the auto-generated
Home / Modules / Tech Debt pages.

### 1. Wiki generator: prose-first prompt (Karpathy alignment)

`libs/wiki/generator.py:_PROMPT_TEMPLATE` previously asked Claude to
write `## Key Components` as a "Bullet list of main files/classes/
functions with one-line descriptions". The new prompt asks for 1-2
paragraphs of connected prose that walk through components and reference
real names inline (`` `Foo.bar` orchestrates X by delegating to `Baz` ``).

Why: Karpathy's April 2026 LLM-Wiki pattern explicitly states that
"the LLM does better with prose than telegraphic bullet points". Wiki
articles are read by an LLM later (via `lvdcp_pack` enrichment, MCP
tools, IDE integrations); connected prose carries more retrievable
signal than a list of symbol names. `Purpose`, `Key Components`,
`Dependencies`, and `Patterns & Decisions` are now all prose. `Known
Issues` keeps bullets — each item is an independent observation, the
one section where bullets actually win.

Existing articles are not auto-rebuilt — they regenerate naturally as
modules go dirty. To force a full rebuild on a project, drop the
`wiki_state` table or run `ctx wiki update --all`.

### 2. Cosmetic fix: no more `modules/README.md.md` double-extension

Until v0.8.65, modules whose source path was already a `.md` file (e.g.
`README.md`, `CHANGELOG.md`, every QA report in X5_BM) ended up with
filenames like `modules/README.md.md` because the writer composed
`f"{module_path}.md"` unconditionally. INDEX.md leaked the cosmetic
ugliness publicly.

`libs/wiki/state.py` now exports `wiki_filename(module_path: str) -> str`
that strips a trailing `.md` (case-insensitive) before appending the
canonical extension. All three call sites (`apps/cli/commands/wiki_cmd.py`,
`apps/agent/wiki_worker.py`, `libs/copilot/orchestrator.py`) were
refactored to use the helper, so the naming rule lives in exactly one
place.

Edge cases covered by tests: `README.md` → `modules/README.md`,
`FOO.MD` → `modules/FOO.md`, `schema.amd` → `modules/schema.amd.md`
(no false-positive strip on partial suffix matches).

### 3. ObsidianPublisher mirrors `.context/wiki/` into `<vault>/Projects/<name>/Wiki/`

`libs/obsidian/publisher.py:ObsidianPublisher.sync_project` now accepts
an optional `wiki_dir: Path | None` parameter. When provided and the
directory exists, the publisher copies:

- `INDEX.md` → `<vault>/Projects/<name>/Wiki/INDEX.md`
- `architecture.md` → `<vault>/Projects/<name>/Wiki/architecture.md`
- `modules/*.md` → `<vault>/Projects/<name>/Wiki/modules/*.md`

`apps/cli/commands/obsidian_cmd.py` always passes
`<project>/.context/wiki` if it exists, so `ctx obsidian sync` (and
nightly `ctx obsidian sync-all`) automatically mirrors the wiki without
new flags.

Drift cleanup: any `*.md` in the destination `Wiki/` tree that we did
*not* just write is deleted (counted in `report.pages_deleted`). This
keeps the vault as the second copy of the truth in `.context/wiki/`,
not an append-only log of every article that ever existed — Karpathy
LLM-Wiki `lint` parity for the vault mirror.

## Why this matters

Before this ship: the wiki was generating bullet-list "Key Components"
sections that read like a symbol dump rather than connected
documentation, and the per-project wiki existed only inside
`.context/wiki/` — invisible from the user's actual knowledge tool
(Obsidian). The artifact was good for `lvdcp_pack` enrichment but
opaque for human review.

After this ship:

- Wiki articles read like prose: 1-2 paragraphs explaining how
  components relate, not a list of names. Better signal for LLM
  retrieval and easier human review.
- Every project synced to Obsidian (`obsidian.enabled: true` +
  `vault_path` configured) gets a `Projects/<name>/Wiki/` folder with
  the full LLM-generated wiki visible alongside the auto-generated
  Home / Modules / Tech Debt pages. Two views, one source of truth in
  `.context/wiki/`.
- The cosmetic `.md.md` double-extension is gone from new INDEX.md
  builds. (Existing files stay until their module goes dirty and
  regenerates.)

## Recovery / migration

- **No schema migration.** `wiki_state.wiki_file` rows from before
  v0.8.66 still resolve correctly — the helper just composes the right
  name on next write.
- **Obsidian users:** if `obsidian.enabled` was already `true`, your
  next sync (manual `ctx obsidian sync` or nightly `sync-all`) will
  populate `<vault>/Projects/<name>/Wiki/` automatically. No flag to
  toggle.
- **Forcing the new prose style on existing articles:** drop the
  `wiki_state` table for that project and re-scan, or run
  `ctx wiki update --all <project>` to regenerate all articles with the
  v0.8.66 prompt.
