## Summary

`v0.7.0` ships the Symbol Timeline Index (spec-010). An append-only SQLite
event store records every symbol transition (added / modified / removed /
renamed) during scans, keyed on `(project_root, symbol_id, commit_sha)`.
Four new MCP tools expose the store through indexed lookups, giving the
assistant first-class answers to temporal questions like *"ą║ąŠą│ą┤ą░ ą▒čŗą╗
čĆąĄą░ą╗ąĖąĘąŠą▓ą░ąĮ X"*, *"čćčéąŠ ą┐čĆąŠą┐ą░ą╗ąŠ ą┐ąŠčüą╗ąĄ v0.6.0"*, *"čćčéąŠ ąĖąĘą╝ąĄąĮąĖą╗ąŠčüčī ą╝ąĄąČą┤čā
v1.0 ąĖ v2.0"*, and *"ą║ą░ą║ąĖąĄ ą┐čāą▒ą╗ąĖčćąĮčŗąĄ čüąĖą╝ą▓ąŠą╗čŗ ąĖčüčćąĄąĘą╗ąĖ ąĮąĄą┤ą░ą▓ąĮąŠ"* ŌĆö without
walking git history blind.

## Shipped

### Data plane

- **Append-only SQLite store** at `~/.lvdcp/symbol_timeline.db`
  (override via `LVDCP_TIMELINE_DB`). WAL-mode, schema-versioned, single
  writer per project. Retention optional via `timeline.retention_days`.
- **Differ + rename detector** (`libs/symbol_timeline/differ.py`,
  `rename_detect.py`) ŌĆö content-hash + qualified-name similarity scoring,
  threshold tunable via `timeline.rename_similarity_threshold` (default
  `0.85`).
- **Scanner wiring** (`libs/scanning/scanner.py`): lazy
  `_maybe_build_default_timeline_sink` reads `timeline.enabled`; zero
  overhead when the flag is off.
- **Release snapshots** (`libs/symbol_timeline/snapshot.py`): tied to
  `git tag`, idempotent insert, checksum-addressed. Tag watcher
  (`libs/gitintel/tag_watcher.py`) promotes new tags into snapshots
  in-process.
- **Reconcile loop** (`libs/symbol_timeline/reconcile.py`): marks events
  orphaned after rebase / `git commit --amend`, surfaced through
  `ctx timeline reconcile`.

### MCP tools

- `lvdcp_when(path, symbol)` ŌĆö full event history + rename lineage for a
  single symbol. Fuzzy-match candidate list when the exact symbol is not
  found.
- `lvdcp_removed_since(path, since)` ŌĆö symbols removed after a git
  reference (tag, SHA, branch, `HEAD~N`). Importance-weighted ranking.
- `lvdcp_diff(path, from_ref, to_ref)` ŌĆö structured diff of
  added / removed / modified / renamed symbols between two refs,
  bucketed with per-bucket `limit`.
- `lvdcp_regressions(path, since, limit)` ŌĆö removed-since restricted to
  symbols with high importance + recent activity, for "what broke?"
  triage.

### CLI

- `ctx timeline enable|disable|status|reconcile|prune|backfill` ŌĆö thin
  Typer wrapper over library functions.

### Claude Code integration

- **Hooks** in `.claude/hooks/` for `PostCommit`, `PostCheckout`, and
  `PostRebase` ŌĆö auto-run reconcile so orphaned events are cleaned up
  without manual intervention.

### Pack enrichment (Layer 4)

- `detect_timeline_markers(query)` ŌĆö EN + RU stem-based regex covering
  *when was*, *since v*, *removed / čāą┤ą░ą╗\*/ą┐čĆąŠą┐ą░[ą╗ą┤]\**, *history of*,
  *diff between*, *renamed from*.
- `enrich_pack_with_timeline(pack, project_root, markers, enabled)` adds
  a `## Timeline facts` section to `ctx pack` output, bounded at **3 KB**
  so enrichment never displaces retrieval results. Silent-skip on
  unreachable store.

### Observability

- **5 Prometheus metrics** (`libs/telemetry/timeline_metrics.py`):
  `symbol_timeline_events_total`, `_query_latency_seconds`,
  `_snapshot_build_seconds`, `_reconcile_orphaned_total`,
  `_sink_errors_total`.
- **structlog enrichment** ŌĆö every timeline-relevant log record carries
  `event_type`, `symbol_id`, `commit_sha`, `duration_ms`.
- **Doctor check** `ctx doctor` ŌåÆ "timeline store": PASS / WARN / FAIL
  with 7-day freshness threshold (spec FR-008).

## Rollout

Default in `TimelineConfig.enabled`: **True** for dev. Operators who want
to hold the feature until they have run a local smoke pass can set
`timeline.enabled: false` in `~/.lvdcp/config.yaml`; re-enable by
flipping back and re-scanning. Store data is append-only and survives
the flag, so soft rollback is free. See the runbook for the 3-step
validation flow.

## Known gaps

- Timeline is populated **prospectively**: events land during scans
  performed with `enabled=true`. There is no backfill from pre-v0.7.0
  history ŌĆö use `ctx timeline backfill` if you want to seed a snapshot
  from an existing `git log`. This is by design per the spec (cost
  envelope).

## Upgrade notes

1. `uv sync` to pull the v0.7.0 package.
2. Run `ctx scan <project>` once; the timeline sink will self-initialize
   at `~/.lvdcp/symbol_timeline.db`.
3. Verify with `ctx timeline status` and `ctx doctor`.
4. Optional: register the Claude Code hooks from `.claude/hooks/` so
   rebases stay reconciled automatically.
